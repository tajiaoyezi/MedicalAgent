## 3. 多租户隔离、RBAC 与 ACL 收口（引用式横切收口，底层判定真值在源头 change）

- [ ] 3.1 收口多租户隔离（复用 c01 `auth-rbac` 隔离基线）：所有查询与写入强制按会话 `tenant_id` 过滤，跨租户读写被拒并不泄露目标资源是否存在，越权尝试写审计（对应「多租户隔离与 RBAC」需求·跨租户场景）
- [ ] 3.2 收口 RBAC（复用 c01 `auth-rbac` 角色模型）：按管理员 / 医生 / 授权审核角色 / 普通用户授予差异化权限，越权操作被拒、无状态变更并记录越权审计（对应 RBAC 角色越权场景）
