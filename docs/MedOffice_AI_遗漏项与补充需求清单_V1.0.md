## 3. 遗漏项 1：文档中心需要完整替代 WPS 云文档

### 3.1 遗漏说明

原 WPS 版大量功能默认依赖 WPS 云文档，包括：

```text
AIMed 上传“我的 WPS 文件”
本地文件上传后转云文档
保存到文字
医学翻译选择 WPS 云文档
WPS 编辑器打开文件
历史任务恢复关联文件
```

独立版如果只做“文件上传 + 下载”，无法完整替代这些能力。

### 3.2 需要补充的产品能力

```text
我的文档
团队文档
应用文档
最近打开
收藏
回收站
版本历史
分享链接
权限设置
文件搜索
文件预览
在线编辑
文件下载
加入知识库
发起 AIMed
发起医学翻译
```

### 3.3 需要补充的技术能力

```text
对象存储：MinIO / S3
文档元数据：PostgreSQL
版本管理：document_versions
权限：document_acl
保存回调：ONLYOFFICE callbackUrl
异步解析：parser worker
RAG 索引：chunk + embedding
审计：document_events
```

### 3.4 建议优先级

```text
P0：我的文档、团队文档、上传、打开、保存、版本、权限
P1：分享、收藏、回收站、加入知识库、文档搜索
P2：高级协作、外链水印、电子签章
```

## 4. 遗漏项 2：ONLYOFFICE Bridge 与医疗 AI 面板协议需要单独定义

### 4.1 遗漏说明

WPS 原方案依赖 WPS 文字组件和右侧 AI 面板的内部通信协议。独立版换成 ONLYOFFICE 后，必须定义新的文档读写桥接协议，否则 AI 面板无法稳定读取当前文档、选区和写回结果。

### 4.2 需要补充的 Bridge API

#### 读取类

```text
getCurrentDocument()
getDocumentId()
getDocumentTitle()
getDocumentType()
getFullText()
getSelectedText()
getCurrentParagraph()
getDocumentOutline()
getCurrentPage()
getComments()
getReferences()
```

#### 写回类

```text
replaceSelection(text)
insertText(position, text)
appendSection(title, content)
insertComment(range, comment)
insertCitation(position, citation)
applyStyle(range, style)
createNewDocument(content, templateId)
createPresentation(slideOutline, templateId)
saveDocument()
```

#### 面板控制类

```text
openAIPanel(command, payload)
closeAIPanel()
runAIPanelSkill(skillName, payload)
streamContentToEditor(content)
```

### 4.3 安全要求

```text
文档 token 校验
用户 token 校验
租户隔离
只读 / 编辑权限校验
插件权限范围控制
AI 写回操作审计
写回前用户确认
```

### 4.4 建议优先级

```text
P0：getSelectedText、getFullText、replaceSelection、insertComment、saveDocument
P1：insertCitation、applyStyle、getDocumentOutline、createNewDocument
P2：复杂排版操作、批量样式写入、协作冲突处理
```

## 5. 遗漏项 3：医疗 AI 写回确认机制需要明确

### 5.1 遗漏说明

医疗文档具有较高责任风险，AI 不应默认直接覆盖用户文档。原 WPS 文档中虽然列出了润色、排版、翻译、校对等能力，但没有明确独立版中的写回责任边界。

### 5.2 需要补充的写回规则

| 操作 | 推荐写回方式 |
|---|---|
| 选区润色 | 展示修改前后对比，用户确认后替换选区 |
| 全文润色 | 生成新文档，不覆盖原文 |
| 校对 | 以批注 / 建议形式插入 |
| 补引用 | 在指定位置插入引用角标和参考文献 |
| 论文排版 | 生成新版本 |
| 医学翻译 | 生成译文副本 |
| 论文转 PPT | 生成新的 PPTX |
| 文档脑图 | 生成脑图视图，不写回原文 |

### 5.3 写回前展示内容

```text
原文
修改后
修改说明
影响范围
按钮：应用到文档 / 生成副本 / 取消
```

### 5.4 建议优先级

```text
P0：选区润色确认、全文生成副本、校对批注
P1：引用插入、排版生成版本、翻译副本
P2：复杂 diff、多人协作冲突确认
```

## 9. 遗漏项 7：AIMed 数据源授权边界需要补充

### 9.1 遗漏说明

当前设计以 PubMed 为 P0 数据源，但原 MVP 方案曾提到万方等中文文献网站内容。中文医学文献数据库通常涉及授权和版权，不能默认承诺接入。

### 9.2 数据源分级建议

| 优先级 | 数据源 | 处理方式 |
|---|---|---|
| P0 | PubMed | 首期默认接入 |
| P0 | 用户上传文档 | 用户授权上传和使用 |
| P0 | 医疗知识库 | 医院自有或授权资料 |
| P1 | 万方 / 知网 / 维普 | 需要商业授权 |
| P1 | 临床指南库 | 需确认版权和使用范围 |
| P1 | 药品说明书 | 需确认数据来源和合规边界 |

### 9.3 PRD 需补充的声明

```text
中文商业文献数据库需完成授权后接入。
未授权时不提供默认检索能力。
用户上传文献仅用于当前用户 / 租户授权范围内的问答和处理。
系统不对未经授权的外部数据库进行爬取。
```

### 9.4 风险等级

```text
高
```

原因：这是版权和数据合规风险，不是技术实现风险。

## 10. 遗漏项 8：AIMed 内翻译和医学翻译模块的分流规则需要补充

### 10.1 遗漏说明

AIMed 的模式关键词中也包含“翻译”，但系统又有独立医学翻译模块。如果不定义分流，用户输入“翻译这篇论文”时系统可能路由错误。

### 10.2 推荐分流规则

| 用户场景 | 推荐模块 |
|---|---|
| 粘贴短文本翻译 | AIMed 学术写作辅助 |
| 选中文档中的一段文字翻译 | 医疗 AI 面板 / AIMed |
| 上传完整文档并说“翻译全文” | 医学翻译模块 |
| 在 ONLYOFFICE 文档内点击“医学翻译” | 医学翻译模块 |
| 深度文献伴读中问“这段讲了什么” | 深度文献伴读 |
| 需要术语库 / 语料库 / 进度 / 下载译文 | 医学翻译模块 |

### 10.3 用户提示

当系统判断更适合医学翻译模块时：

```text
检测到您需要翻译完整文档。建议进入「医学翻译」模块，可支持术语库、排版保持、进度查看和译文下载。
```

## 26. 最终判断

当前独立版方案已经覆盖原 WPS 医疗版的一级功能。此前识别的 18 类二级功能和边界已按处理状态并入主 PRD、调整口径或延期。若后续重新拆 phase / task，只读取主 PRD，不读取本文档作为执行输入。

```text
本文档用途：
- 审查主 PRD 是否遗漏历史风险项。
- 追踪哪些遗漏项已处理。
- 解释为什么数字员工延期。
- 避免后续把旧 P0/P1 建议误当成执行范围。
```

因此，下一步应以主 PRD 为唯一输入生成 phase、task、ADR 和验收用例；本文档仅作为审查追溯材料保留。
